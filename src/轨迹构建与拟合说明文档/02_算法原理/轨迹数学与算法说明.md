# TRAJECTORY_MATH_GUIDE（面向 ROS2 迁移）

## 1. 轨迹链路总览

当前工程的轨迹生成与执行链路可以抽象为：

`点云特征 -> 三段轨迹 pathBase -> 末端位姿序列 T_target_seq -> 关节接近路径 qApproach -> IK跟踪执行 qExec`

对应模块关系：

1. 感知：`estimate_bucket_from_pcd` 输出 `bucketFeatures`。
2. 轨迹：`fit_three_segment_parabola` 输出 `pathBase`（笛卡尔离散路径）。
3. 姿态：`build_target_pose_seq` 将 `pathBase` 转为 `T_target_seq`。
4. 接近规划：`plan_rrtstar_joint_path` 或 `plan_dprrtstar_joint_path_cuda` 输出 `qApproach`。
5. 执行：`run_ik_and_animation` 逐点 IK + 连续段连接，输出 `qExec`。

这套链路后续迁移到 ROS2 时，建议保持“先几何，后运动学，最后控制”的顺序，不要一次性重构。

## 2. 输入/输出定义

### 2.1 `bucketFeatures`（感知输出）

核心字段（代码实际使用）：

- `topPoint`：桶口参考点（世界坐标）。
- `topRadius`：桶口半径。
- `bottomRadius`：桶底半径。
- `depth`：桶深。
- `maxRadius`：感知最大半径。
- `digDirection`：挖掘主方向（通常来自 PCA 水平主轴）。
- `cloudPoints`（可选）：用于最小二乘轮廓拟合。

### 2.2 `trajParams`（轨迹参数）

高影响字段：

- `entrySpanFactor/midSpanFactor/exitSpanFactor`
- `entryDepthRatio/targetDeepDepthRatio`
- `midSlopeAtKnot1`
- `maxCutAngleDeg`
- `topOuterOffsetRatio`
- `startFinishLiftRatio`
- `execFirstMode`
- `forceBaseXDir`
- `reverseFitZAxis`

### 2.3 `postureParams`（姿态参数）

- `attackDeg`
- `assemblyDeg`
- `flipToolZ`

### 2.4 关键输出

- `pathBase (N x 3)`：离散轨迹点（世界坐标）。
- `T_target_seq (4 x 4 x N)`：每个轨迹点对应末端目标位姿。
- `qApproach (M x J)`：起始到入轨点的关节路径。
- `qExec (K x J)`：最终执行关节序列。

## 3. 三段轨迹生成算法（`fit_three_segment_parabola`）

### 3.1 局部坐标系构造

在桶口附近构造拟合平面坐标：

- `xAxis`：
  - 若 `forceBaseXDir=true`，固定为世界 X 轴。
  - 否则取 `digDirection` 的水平分量归一化。
- `zAxis`：
  - 正常为 `[0,0,1]`。
  - 若 `reverseFitZAxis=true`，取 `[0,0,-1]`。
- `yAxis = normalize(cross(zAxis, xAxis))`

该设计保证轨迹主要在 `x-z` 平面变化，`y` 仅做小偏置。

### 3.2 四个关键横坐标与纵坐标

定义 4 个关键点（局部 x 方向）

- `x0`：入桶前起点（负向外侧）
- `x1`：中段过渡点
- `x2`：最深点附近（通常设为 0）
- `x3`：出桶终点（正向外侧）

并定义对应高度：

- `z0,z3`：抬高到桶口上方（`startFinishLiftRatio * depth`）
- `z1`：入桶深度（`entryDepthRatio * depth`）
- `z2`：目标最深深度（固定为 `targetDeepDepthRatio * depth`）

### 3.3 分段曲线求解

三段函数分别为 `seg1/seg2/seg3`，标准形式：

`z(x)=ax^2+bx+c`

代码中用线性方程组按“端点 + 端点斜率”求解二次项：

- `solveQuadWithEndSlope`：给定 `(xa,za),(xb,zb), slope@xb`
- `solveQuadWithStartSlope`：给定 `(xa,za),(xb,zb), slope@xa`
- `solveLineBetweenPoints`：执行优先模式下，中段可退化为直线

### 3.4 安全约束与执行优先约束

1. 最大切削角约束：

- `maxSlope = tan(maxCutAngleDeg)`
- 限制关键斜率与离散点斜率：`|dz/dx| <= maxSlope`

2. 最深点固定约束：

- 强制 `z2 = topZ - targetDeepDepthRatio * depth`

3. 执行优先迭代限幅（`enforceSlopeWithExecPriority`）：

- 对离散点做前向/后向两遍限幅迭代，避免局部过陡。
- 迭代期间锁定起终点和最深点。

伪公式：

```text
for i = 2..N:
  dz = z[i]-z[i-1]
  dz = clamp(dz, -maxSlope*dx, +maxSlope*dx)
```

### 3.5 最小二乘轮廓融合（`tryLeastSquareProfile`）

当 `bucketFeatures.cloudPoints` 可用时：

1. 投影点云到局部 `x-z`。
2. 用 `z = ax^2 + bx + c` 做最小二乘拟合。
3. 对关键高度 `z1` 做融合修正（代码中为 0.5 混合）。

用途：让轨迹更贴桶体真实剖面。  
限制：点数不足、带宽筛选失败时会自动放弃，回退到规则几何。

## 4. 笛卡尔轨迹到末端姿态序列（`build_target_pose_seq`）

### 4.1 切向量与局部正交基

对每个轨迹点 `p_i`：

- 用前后差分估计切向量 `t_i`。
- 设工具 `yAxis = normalize(t_i)`（工具 y 跟随轨迹方向）。
- 给定参考轴 `xRef`（默认 `[0,1,0]`），计算
  - `zAxis = normalize(cross(xRef, yAxis))`
  - `xAxis = normalize(cross(yAxis, zAxis))`

### 4.2 `flipToolZ` 与连续性防翻转

- 若 `flipToolZ=true`，取 `zAxis=-zAxis`。
- 与上一帧比较，若 `dot(xAxis,prevX)<0` 或 `dot(zAxis,prevZ)<0`，整体翻转 `(x,z)`，避免姿态突然 180° 反转。

### 4.3 攻角/装配角补偿

补偿角：

`thetaComp = deg2rad(attackDeg + assemblyDeg)`

绕局部 y 轴应用轴角旋转：

`Rcomp = axis_angle_rotm([0,1,0], thetaComp)`

最终：

`Rtar = Rnom * Rcomp`，再与位置拼成 `T_target_seq(:,:,i)`。

## 5. 关节空间规划算法

## 5.1 RRT*（`plan_rrtstar_joint_path`）

基础流程：

1. 在关节限位内采样 `qRand`（含 `goalBias` 目标偏置）。
2. 最近邻：找 `qNear`。
3. `steer`：沿 `qNear->qRand` 走一步 `stepSize` 得 `qNew`。
4. 碰撞边检查：`edgeCollisionFree`。
5. 邻域选父：半径 `nearRadius` 内最小代价父节点。
6. `rewire`：邻域重连降低总代价。
7. 近目标且连边无碰撞则接入 `qGoal`。
8. 末端 `shortcut` 平滑路径。

### 5.2 DP-RRT*（`plan_dprrtstar_joint_path_cuda`）

在 RRT* 上增加三种动态机制：

1. 动态目标偏置 `Pg`：

`Pg = max(Pg_min, Pg_init * exp(-decayRate * failCount))`

2. 动态引力融合 `rho`：

`rho = clamp(rho_init * exp(-decayRate * failCount), 0, 1)`

`v_new = normalize((1-rho)*v_rand + rho*v_goal)`

3. 自适应步长 `lambda(d_min)`：

- `d_min` 综合关节限位余量和历史碰撞样本最近距离。
- `lambda` 在 `[lambdaMin, lambdaMax]` 内按安全裕度映射。

`lambda = lambdaMin + (lambdaMax-lambdaMin)*ratio(d_min)`

额外机制：

- 边碰撞时记录 `qHit` 到 `collisionSamples`。
- 失败计数增长，成功时按 `failRecovery` 回退。

### 5.3 GPU 使用边界

当前实现中 GPU 仅用于：

- 最近邻距离计算
- 半径邻域查询

碰撞检测仍在 CPU 执行。迁移 ROS2 时也建议保持此边界，先稳后快。

## 6. IK 执行与退化策略（`run_ik_and_animation`）

### 6.1 主 IK 校验

每个目标位姿先求 `qTry`，再做三重校验：

1. 碰撞：`collisionFcn(qTry)`
2. 位置误差：`posErr <= posTol`
3. 姿态误差：`oriErr <= oriTolDeg`

姿态误差通过 `rotm_geodesic_deg` 计算。

### 6.2 fallback：保持姿态、缩短位移

若主 IK 失败：

- 保持目标姿态 `Rtar` 不变。
- 位置从 `pPrev` 朝 `pTar` 用 `alpha=[0.9,0.75,...,0.1]` 递减尝试。
- 找到第一个满足“无碰撞 + 姿态误差可接受”的解。

### 6.3 段连接与保底

从上一关节点到下一关节点的连接：

1. 先线性插值检查。
2. 若碰撞，尝试局部 `RRT*`。
3. 若仍失败，用 `hold`（冲突采样点回退为段起点）保证连续执行不中断。

这是实际机械臂中非常关键的“不中断策略”。

## 7. ROS2 映射设计（核心）

## 7.1 节点拆分建议

1. `perception_node`
- 订阅点云。
- 发布 `bucket_features`（等价 `bucketFeatures`）。

2. `trajectory_node`
- 订阅 `bucket_features` 与轨迹参数。
- 发布 `path_base`、`target_pose_seq`。

3. `planner_node`
- 订阅 `target_pose_seq` 与当前关节状态。
- 发布 `q_approach`（接近段）与/或整体关节路径。

4. `executor_node`
- 执行 IK 跟踪/轨迹时间化。
- 对接 `joint_trajectory_controller` 或 MoveIt2 执行接口。
- 发布执行反馈与异常状态。

## 7.2 话题与消息建议

- `/bucket/features`：自定义消息（top point/radius/depth/direction）。
- `/trajectory/path_base`：`geometry_msgs/PoseArray` 或 `Point[]`。
- `/trajectory/target_poses`：`PoseArray`。
- `/planning/q_approach`：`sensor_msgs/JointState[]` 或自定义数组消息。
- `/execution/feedback`：执行进度、误差、fallback/hold 次数。

## 7.3 控制周期与时间参数化

MATLAB 轨迹是“按点离散”。到 ROS2 实机必须加时间：

1. 先按弧长或关节增量重采样。
2. 施加速度/加速度约束（推荐 MoveIt2 Time Parameterization）。
3. 保证控制周期稳定（如 100Hz 或控制器要求频率）。

## 8. 参数影响表（含默认值与调参方向）

| 参数 | 默认值 | 增大效果 | 减小效果 | 主要风险 |
|---|---:|---|---|---|
| `targetDeepDepthRatio` | 0.15 | 挖得更深 | 挖得更浅 | 深处碰撞/不可达 |
| `entryDepthRatio` | 0.20 | 入桶更深 | 入桶更浅 | 前段 IK 压力 |
| `midSlopeAtKnot1` | -0.20 | 中段更陡（绝对值增） | 更平缓 | 切削角超限 |
| `maxCutAngleDeg` | 30 | 允许更陡斜率 | 更保守 | 过大易抖动/碰撞 |
| `topOuterOffsetRatio` | 0.125 | 起终点更外 | 更靠近桶口 | 外侧过远影响可达 |
| `startFinishLiftRatio` | 0.25 | 起终点更高 | 更低 | 低位易擦碰桶沿 |
| `nPts` | 90 | 更平滑细腻 | 更稀疏 | 过大增加计算量 |
| `attackDeg` | -12 | 姿态更前倾 | 更直立 | 过大易超姿态约束 |
| `assemblyDeg` | 0 | 装配补偿更强 | 更弱 | 标定误差放大 |
| `goalBiasInit` | 0.28 | 更快朝目标 | 探索性增强 | 易陷局部障碍 |
| `lambdaMax` | 0.24 | 扩展更快 | 更稳但慢 | 过大跳过窄通道 |
| `lambdaMin` | 0.08 | 障碍附近步长更大 | 更谨慎 | 过小规划变慢 |
| `dpDecayRate` | 0.50 | 动态参数衰减更快 | 衰减更慢 | 早期/后期策略失衡 |
| `useGPU` | true | 加速距离计算 | CPU 回退 | 环境依赖问题 |

## 9. 失败模式与调试清单（实机优先）

### 9.1 不可达 / IK 大误差

看指标：`posErr`、`oriErrDeg`、`solInfo.Status`。  
先调：减小 `targetDeepDepthRatio`、减小 `|attackDeg|`、增大 `startFinishLiftRatio`。  
建议日志：每点打印 `i, posErr, oriErr, colliding`。

### 9.2 碰撞密集

看指标：`firstHitBody`、`collideCount`、`localRrtFailTotal`。  
先调：减小深度、增大外偏、核对白名单 body。  
建议日志：碰撞点关节向量摘要、hit type。

### 9.3 规划超时

看指标：`iterUsed`、`nearGoalDist`、`failCountFinal`、`elapsedSec`。  
先调：`maxIter`、`goalBiasInit`、`nearRadius`、`lambdaMax`。  
建议日志：每 `logEvery` 输出 `Pg/rho/lambda/dMin`。

### 9.4 GPU 回退或失败

看指标：`gpu.enabled`、`gpu.reason`。  
先调：`forceGPU=false` 确认可回退，再排查 CUDA 环境。  
建议日志：记录主规划与 baseline 耗时、路径长度对比。

### 9.5 局部抖动/姿态翻转

看指标：相邻姿态轴点乘、轨迹曲率、关节速度尖峰。  
先调：增大 `nPts`，限制姿态变化率，检查 flip 连续性逻辑。  
建议日志：输出 `dot(x_i,x_{i-1})`、`dot(z_i,z_{i-1})`。

### 9.6 执行中断/无进展

看指标：`timeoutStage`、`nHold`、`consecutiveNoProgress`。  
先调：放宽 `maxConsecutiveNoProgress`，适度增大 `localRrtMaxIter`，调低轨迹激进度。  
建议日志：每段输出 `segProgress/localRrtTried/localRrtSolved`。

## 10. MATLAB 到 ROS2 的最小迁移路线图

### 阶段A：离线一致性复现

1. 用同一组 `bucketFeatures + trajParams + postureParams` 在 ROS2 节点复现 `pathBase`。
2. 再复现 `T_target_seq`，与 MATLAB 结果逐点比对（位置误差、姿态误差）。

验收：轨迹几何形状与姿态连续性一致。

### 阶段B：接入 MoveIt2 规划

1. 将 `qApproach` 规划交给 MoveIt2（或保留自定义 RRT* 对比）。
2. 统一碰撞模型与关节限位来源（URDF/SRDF）。

验收：可稳定到达入轨点，规划成功率和耗时可接受。

### 阶段C：执行闭环上线

1. 对 `T_target_seq` 或转换后的关节轨迹做时间参数化。
2. 下发到 `joint_trajectory_controller`，加入监控与中断恢复策略。
3. 保留 fallback/hold 思想，但在实机必须增加安全联锁（急停、软限位）。

验收：完整动作可重复执行，异常可控、日志可追溯。
