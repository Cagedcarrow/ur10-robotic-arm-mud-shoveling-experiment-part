# ROS 2 核心文件写法

## 一句话理解

ROS 2 包里最常见的核心文件是：

- `package.xml`：声明包信息和依赖
- `setup.py` 或 `CMakeLists.txt`：声明如何构建和安装
- `launch/*.launch.py`：声明如何启动
- `config/*.yaml`：声明参数

如果这几类文件写清楚，包的结构就基本稳定了。

## 1. package.xml

### 作用

`package.xml` 是包的元信息文件，相当于“包的身份证”。

它至少需要说明：

- 包名
- 版本
- 描述
- 维护者
- 许可证
- 依赖
- 构建类型

### Python 包常用示例

```xml
<?xml version="1.0"?>
<?xml-model href="http://download.ros.org/schema/package_format3.xsd" schematypens="http://www.w3.org/2001/XMLSchema"?>
<package format="3">
  <name>my_python_package</name>
  <version>0.0.1</version>
  <description>example package</description>
  <maintainer email="user@example.com">User</maintainer>
  <license>Apache-2.0</license>

  <buildtool_depend>ament_python</buildtool_depend>

  <exec_depend>rclpy</exec_depend>
  <exec_depend>std_msgs</exec_depend>

  <export>
    <build_type>ament_python</build_type>
  </export>
</package>
```

### C++ 包常用示例

```xml
<buildtool_depend>ament_cmake</buildtool_depend>
<build_depend>rclcpp</build_depend>
<exec_depend>rclcpp</exec_depend>
<exec_depend>std_msgs</exec_depend>

<export>
  <build_type>ament_cmake</build_type>
</export>
```

## 2. setup.py

### 作用

`setup.py` 主要用于 Python 包，定义：

- 哪些 Python 模块需要安装
- 哪些 launch / config / rviz 文件需要安装
- 哪些节点命令要暴露给 `ros2 run`

### 常见模板

```python
from setuptools import setup
from glob import glob

package_name = 'my_pkg'

setup(
    name=package_name,
    version='0.0.1',
    packages=[package_name],
    data_files=[
        ('share/ament_index/resource_index/packages', ['resource/' + package_name]),
        ('share/' + package_name, ['package.xml']),
        ('share/' + package_name + '/launch', glob('launch/*.launch.py')),
        ('share/' + package_name + '/config', glob('config/*.yaml')),
    ],
    install_requires=['setuptools'],
    zip_safe=False,
    maintainer='User',
    maintainer_email='user@example.com',
    description='example package',
    license='Apache-2.0',
    entry_points={
        'console_scripts': [
            'my_node = my_pkg.my_node:main',
        ],
    },
)
```

### 最重要的两个点

#### `data_files`

决定 launch、配置文件、RViz 配置等是否能在安装后被找到。

#### `entry_points`

决定 `ros2 run my_pkg my_node` 能不能工作。

## 3. CMakeLists.txt

### 作用

C++ 包通常使用 `CMakeLists.txt` 管理构建。

### 最小示例

```cmake
cmake_minimum_required(VERSION 3.8)
project(my_cpp_pkg)

find_package(ament_cmake REQUIRED)
find_package(rclcpp REQUIRED)
find_package(std_msgs REQUIRED)

add_executable(my_node src/my_node.cpp)
ament_target_dependencies(my_node rclcpp std_msgs)

install(TARGETS my_node
  DESTINATION lib/${PROJECT_NAME})

install(DIRECTORY launch config
  DESTINATION share/${PROJECT_NAME})

ament_package()
```

### 最重要的几部分

- `find_package(...)`：找依赖
- `add_executable(...)`：声明可执行文件
- `ament_target_dependencies(...)`：链接依赖
- `install(...)`：声明安装规则
- `ament_package()`：导出包

## 4. launch 文件

### 作用

launch 文件用来描述：

- 启动哪些节点
- 给节点传哪些参数
- 是否加载 RViz、robot_state_publisher 等

### 常见写法

```python
from launch import LaunchDescription
from launch_ros.actions import Node


def generate_launch_description():
    return LaunchDescription([
        Node(
            package='my_pkg',
            executable='my_node',
            output='screen',
        )
    ])
```

### 什么时候需要 launch

- 需要同时启动多个节点
- 需要加载参数文件
- 需要加载 RViz
- 需要启动机器人描述、TF、传感器等组合场景

## 5. 参数文件 YAML

### 作用

把节点参数从代码中抽离出来，便于修改和复用。

### 示例

```yaml
my_node:
  ros__parameters:
    publish_rate: 10.0
    frame_id: base_link
    use_sim_time: false
```

### launch 中使用

```python
Node(
    package='my_pkg',
    executable='my_node',
    parameters=['config/params.yaml'],
)
```

## 6. 节点文件的基本结构

### Python 节点模板

```python
import rclpy
from rclpy.node import Node


class MyNode(Node):
    def __init__(self):
        super().__init__('my_node')
        self.get_logger().info('node started')


def main(args=None):
    rclpy.init(args=args)
    node = MyNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

### 至少要有的部分

- 导入依赖
- 定义节点类
- 在 `__init__` 中完成初始化
- 写 `main()` 函数
- 在脚本末尾调用入口

## 7. 最常见的错误

### `package.xml` 问题

- 包名和目录名不一致
- 缺少 `buildtool_depend`
- 缺少实际使用到的运行依赖

### `setup.py` 问题

- `entry_points` 路径写错
- 忘记安装 `launch/` 或 `config/`
- `packages=[...]` 没包含真正的 Python 模块

### `CMakeLists.txt` 问题

- `find_package` 漏写依赖
- `install(...)` 漏掉 launch 或 config
- 缺少 `ament_package()`

## 8. 推荐检查顺序

每次新建或排错一个包时，优先按这个顺序检查：

1. `package.xml` 依赖齐不齐
2. `setup.py` 或 `CMakeLists.txt` 安装规则对不对
3. 节点入口有没有配好
4. launch 是否引用了正确的节点名和参数文件
5. 编译后有没有重新 `source install/setup.bash`

## 9. 快速记忆

```text
package.xml 管身份和依赖
setup.py / CMakeLists.txt 管安装和构建
launch 管启动方式
yaml 管参数
节点文件管真正的业务逻辑
```
