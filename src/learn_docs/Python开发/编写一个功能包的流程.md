# 编写一个 ROS 2 功能包的流程

## 一句话理解

写一个功能包最稳的方式是：先用命令生成骨架，再补节点代码，再补 launch 和配置，最后检查依赖、安装规则和编译流程。

## 1. 先确定包类型

最常见的两类：

- Python 包：`ament_python`
- C++ 包：`ament_cmake`

如果你主要写 Python 节点，优先用 Python 包。

## 2. 创建包骨架

### Python 包

```bash
cd ~/ros2_ws/src
ros2 pkg create my_python_package \
  --build-type ament_python \
  --dependencies rclpy std_msgs \
  --license Apache-2.0
```

### C++ 包

```bash
cd ~/ros2_ws/src
ros2 pkg create my_cpp_package \
  --build-type ament_cmake \
  --dependencies rclcpp std_msgs \
  --license Apache-2.0
```

## 3. 看懂自动生成的文件

以 Python 包为例，常见结构如下：

```text
my_python_package/
├── my_python_package/
│   ├── __init__.py
│   └── my_node.py
├── resource/
├── test/
├── package.xml
├── setup.py
├── setup.cfg
└── README.md
```

这些文件里：

- 自动生成但通常需要修改：`package.xml`、`setup.py`
- 你必须手动写逻辑：节点代码、launch、配置文件
- 一般不用改：`__init__.py`、`setup.cfg`

## 4. 先写最小可运行节点

### Python 示例

```python
import rclpy
from rclpy.node import Node


class MyNode(Node):
    def __init__(self):
        super().__init__('my_node')
        self.get_logger().info('node started')


def main(args=None):
    rclpy.init(args=args)
    node = MyNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()
```

## 5. 修改 package.xml

最小 Python 包常见内容：

```xml
<buildtool_depend>ament_python</buildtool_depend>
<exec_depend>rclpy</exec_depend>
<exec_depend>std_msgs</exec_depend>

<export>
  <build_type>ament_python</build_type>
</export>
```

检查点：

- 包名是否正确
- 描述、维护者、许可证是否填好
- import 到的依赖是否都声明了

## 6. 修改 setup.py

至少要保证两件事：

1. 节点能被 `ros2 run` 调到
2. launch / config 等资源能安装进去

### 最小示例

```python
entry_points={
    'console_scripts': [
        'my_node = my_python_package.my_node:main',
    ],
}
```

如果有 launch 和参数文件，还要在 `data_files` 里加上：

```python
('share/' + package_name + '/launch', glob('launch/*.launch.py')),
('share/' + package_name + '/config', glob('config/*.yaml')),
```

## 7. 按需补目录和文件

如果包稍微复杂一点，通常还需要补这些目录：

```bash
mkdir -p launch config
```

### launch 文件

```python
from launch import LaunchDescription
from launch_ros.actions import Node


def generate_launch_description():
    return LaunchDescription([
        Node(
            package='my_python_package',
            executable='my_node',
            output='screen',
        )
    ])
```

### 参数文件

```yaml
my_node:
  ros__parameters:
    use_sim_time: false
```

## 8. 编译

### Python 开发推荐

```bash
cd ~/ros2_ws
source /opt/ros/humble/setup.bash
colcon build --packages-select my_python_package --symlink-install
source install/setup.bash
```

### C++ 包

```bash
cd ~/ros2_ws
source /opt/ros/humble/setup.bash
colcon build --packages-select my_cpp_package
source install/setup.bash
```

## 9. 运行

### 运行单节点

```bash
ros2 run my_python_package my_node
```

### 用 launch 运行

```bash
ros2 launch my_python_package demo.launch.py
```

## 10. 常见场景下需要哪些文件

### 最小化单节点包

至少需要：

- 节点代码
- `package.xml`
- `setup.py` 或 `CMakeLists.txt`

### 多节点 + launch

额外需要：

- `launch/*.launch.py`
- 可能还需要 `config/*.yaml`

### 自定义消息 / 服务 / 动作

额外需要：

- `msg/*.msg`
- `srv/*.srv`
- `action/*.action`
- 对应的接口依赖和生成配置

## 11. 最常见的坑

### `ros2 run` 找不到节点

检查：

- `setup.py` 的 `entry_points`
- 是否重新编译
- 是否 `source install/setup.bash`

### launch 找不到文件

检查：

- launch 文件是否放在 `launch/`
- `setup.py` / `CMakeLists.txt` 是否安装了它

### import 报错

检查：

- `package.xml` 是否声明依赖
- 环境是否 source 正确

## 12. 推荐工作节奏

每次新建包时，建议按这个顺序走：

1. 创建包骨架
2. 先写一个最小节点跑通
3. 再补 launch 和配置
4. 再增加复杂逻辑
5. 每走一步都编译并验证一次

这样最不容易在最后阶段一起爆一堆错。
