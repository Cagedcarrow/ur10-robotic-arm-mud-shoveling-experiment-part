# UR10 实机验证 GUI 使用说明

## 1. 文件位置
- GUI 脚本：`/root/ur10_ws/src/ur10_real_comm/scripts/ur10_real_verify_gui.py`
- 本说明：`/root/ur10_ws/src/ur10_real_comm/scripts/UR10_实机验证GUI使用说明.md`

## 2. 启动方式
在终端执行：

```bash
cd /root/ur10_ws/src/ur10_real_comm
python3 scripts/ur10_real_verify_gui.py
```

## 3. 主要能力
- 一键启动原有 `run_full_verify_wsl.sh` 流程。
- 快速预检查：参数、网段、路由、Ping、TCP 端口。
- 高级网络诊断：多轮 Ping/TCP 时延统计。
- 可选协议探测（参考 `ur10_server_.py` 的字节解析方式）。
- 日志导出、参数保存/加载。

## 4. 关键参数说明
- `主机IP（Windows侧）`：UR 回连的目标主机 IP（例如 `192.168.56.1`）。
- `机器人IP`：UR 控制器 IP。
- `网段（CIDR）`：例如 `192.168.56.0/24`。
- `TCP端口`：默认 `8001`（与你的 `ur10_server_.py` 一致）。
- `TCP超时`：TCP 与协议探测超时时间（秒）。
- `run_full_verify_wsl.sh 路径`：默认会自动尝试常见路径。

## 5. 协议探测模式（重要）
`ur10_server_.py` 中按 `data[1:2]` 识别命令，因此 GUI 提供以下模式：

- `safe_invalid`（默认，推荐）：发送未定义命令（`00 90`），通常用于确认“服务在线且可回包”，不会走 A1/A2/A3 业务流程。
- `A1` / `A2` / `A3`：发送真实业务命令字，可能触发机械臂动作。
- `custom_hex`：自定义十六进制字节流，例如 `00 a1`。

建议：
- 日常联调优先使用 `safe_invalid`。
- 仅在现场确认安全条件后再使用 `A1/A2/A3`。

## 6. 推荐使用流程
1. 填写 IP、网段、端口、脚本路径。
2. 点击 `快速预检查`，确认基础网络通过。
3. 如需更详细诊断，点击 `高级网络诊断`。
4. 检查无阻断后点击 `开始验证`。
5. 若流程中断，可使用 `发送确认服务` 或 `查询控制器状态` 辅助定位。
6. 完成后点击 `导出日志` 保存记录。

## 7. 常见问题
- `脚本不存在`：请确认 `run_full_verify_wsl.sh` 路径是否正确。
- `TCP端口不可达`：检查 UR 端服务是否已启动、IP/端口是否匹配、防火墙是否放行。
- `ping 失败但 TCP 偶尔成功`：可能是网络策略屏蔽 ICMP，可优先看 TCP 检测结果。
- `协议探测超时`：服务未运行、端口错误，或业务命令执行耗时超出超时设置。

## 8. 配置文件
GUI 会自动保存参数到：

```text
~/.ur10_real_verify_gui.json
```

你可通过 `保存参数` / `加载参数` 管理不同场景配置。
