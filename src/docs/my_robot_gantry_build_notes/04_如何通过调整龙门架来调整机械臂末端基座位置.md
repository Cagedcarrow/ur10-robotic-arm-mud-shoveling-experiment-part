# 如何通过调整龙门架来调整机械臂末端基座位置

## 这篇文档解决什么问题

这篇文档专门讲实际操作：

- 我应该调哪个量？
- 调完之后，机械臂哪里会变？
- 这是在调末端执行器，还是在调机械臂基座？

先给最重要的结论：

你在这里调整的是机械臂基座 `ur10` 相对世界坐标的位置，而不是直接拖动机械臂 6 个旋转关节。

## 什么叫“机械臂基座位置”

在本项目中，机械臂挂在龙门架下方。

所以机械臂不是直接固定在地面上的，而是固定在：

```text
gantry_z_carriage_link -> ur_mount_link -> ur10
```

这里的 `ur10` 就是机械臂基座 link。

当龙门架移动时，机械臂基座会整体跟着移动，进而带动整台机械臂的工作空间一起移动。

## 这和“机械臂末端位置”不是一回事

要区分两个概念：

### 1. 机械臂基座位置

这是机械臂根部在世界中的位置。

改变方式：

- 改龙门架 X/Y/Z 三轴

效果：

- 整台机械臂整体平移

### 2. 机械臂末端位置

这是 `shovel_tip` 这样的末端点在空间中的位置。

改变方式：

- 改机械臂关节
- 或者在基座移动后，末端也会被动跟着整体移动

效果：

- 末端相对于基座的位置变化，通常来自机械臂关节运动

所以你这次问的“通过调整龙门架来调整机械臂末端基座的位置”，更准确地说，是：

通过调整龙门架来调整机械臂基座的位置，从而改变机械臂末端整体可达空间。

## 三个龙门架关节分别控制什么

### `gantry_x_joint`

含义：

- 控制机械臂基座沿世界 X 方向平移。

直观理解：

- 把整台机械臂向左或向右搬动。

### `gantry_y_joint`

含义：

- 控制机械臂基座沿世界 Y 方向平移。

直观理解：

- 把整台机械臂向前或向后搬动。

### `gantry_z_joint`

含义：

- 控制机械臂基座沿世界 Z 方向平移。

直观理解：

- 把整台机械臂升高或降低。

注意：

- 在当前配置里 Z 的数值通常是相对顶梁参考位置的偏移。
- 因为安装方式是从上往下挂，所以你会看到它的初始值和范围里有负数。

## 调龙门架后，机械臂会发生什么

调龙门架不是只动某一个关节，而是会让以下这些东西一起变：

- `ur10` 基座位置变化
- 整台机械臂的世界坐标位置变化
- `shovel_tip` 的世界坐标位置也随之整体变化
- 机械臂的工作空间整体平移

这件事非常重要，因为它意味着：

你可以先通过龙门架把机械臂搬到“更合适的作业区域”，再让 MoveIt 去做机械臂自身的精细动作规划。

## 操作方式 1：通过配置初始值调整

如果你希望“启动时机械臂基座就在一个新的默认位置”，最直接的方式是改：

`my_robot/config/gantry_config.json`

关键字段：

```json
{
  "initial": {
    "x": 0.0,
    "y": 0.0,
    "z": -0.6
  }
}
```

你可以这样理解：

- `initial.x`
  - 启动后 X 轴默认偏移
- `initial.y`
  - 启动后 Y 轴默认偏移
- `initial.z`
  - 启动后 Z 轴默认偏移

改完之后重新启动：

```bash
source /opt/ros/humble/setup.bash
source /root/ur10_ws/install/setup.bash
ros2 launch my_robot sim_planning_gantry.launch.py gui:=true start_rviz:=true
```

启动完成后，龙门架初始化程序会把三轴移动到 JSON 中指定的初始位置。

## 操作方式 2：通过命令行脚本调整

如果你想在系统运行中直接给龙门架发位置目标，可以使用 `gantry_control.py` 对应的可执行程序。

本质上它会向：

```text
/gantry_trajectory_controller/follow_joint_trajectory
```

发送一条三轴轨迹目标。

它控制的 joints 是：

- `gantry_x_joint`
- `gantry_y_joint`
- `gantry_z_joint`

这意味着你可以用脚本方式把机械臂基座移动到任意允许范围内的位置。

这个方式适合：

- 调试
- 写自动化流程
- 固定点位测试

## 操作方式 3：通过 RViz2 交互拖拽调整

这是最直观的一种方式。

本项目里提供了 `gantry_rviz_control.py`，它会在 RViz2 中创建一个 Interactive Marker。

你拖动这个 Marker 时，程序会做几件事：

1. 读取 Marker 在世界坐标中的位置。
2. 根据龙门架基准位置，换算出对应的 X/Y/Z 三轴目标值。
3. 自动把目标夹紧到允许范围内。
4. 通过 `gantry_trajectory_controller` 发送轨迹命令。

它的内部换算逻辑是：

- Marker 的世界坐标
- 减去龙门架固定基准位置
- 得到 `gantry_x_joint`、`gantry_y_joint`、`gantry_z_joint` 的目标值

这也是为什么：

在 RViz2 里你看到的是“拖一个空间控制点”，但实际控制的是龙门架三轴关节。

## 如何判断我调的是“基座”而不是“末端”

看现象最容易判断。

如果你调龙门架后看到：

- 整台机械臂一起平移
- 机械臂自身的姿态没有明显变化

那你调的是基座位置。

如果你调的是机械臂关节，通常会看到：

- 基座不动
- 手臂姿态变化
- 末端相对基座运动

## `gantry_base_height` 在实际操作里怎么理解

`gantry_base_height` 决定了龙门架上层结构的参考高度。

你可以把它理解成：

- 龙门架顶梁的结构高度
- X 轴滑台默认挂载高度

它不是一个“控制命令”，更像是机械结构参数。

如果你改它：

- 相当于整个龙门架结构高度变化
- 会间接影响机械臂基座的可达高度范围

## 关节限位为什么重要

三轴移动不是无限的，所以配置里有这些限制：

- `gantry_x_min/max`
- `gantry_y_min/max`
- `gantry_z_min/max`

它们的作用是：

- 防止目标越界
- 让控制器和交互脚本都只在允许范围内工作

在 RViz 交互脚本里，拖拽目标也会被夹紧到这个范围。

这意味着：

你即使把 Marker 拖得很远，最终发送给控制器的也是一个合法目标。

## 一个典型使用流程

下面给一个很常见的工作流程。

### 步骤 1：启动系统

```bash
source /opt/ros/humble/setup.bash
source /root/ur10_ws/install/setup.bash
ros2 launch my_robot sim_planning_gantry.launch.py gui:=true start_rviz:=true
```

### 步骤 2：先看机械臂基座当前在什么位置

观察 Gazebo 和 RViz2 中：

- 龙门架滑台位置
- 机械臂基座挂载位置

### 步骤 3：移动龙门架

可以选一种方式：

- 改 `gantry_config.json` 后重启
- 用脚本发送三轴目标
- 在 RViz2 里拖交互 Marker

### 步骤 4：确认基座已经搬到目标区域

这一步要观察：

- 机械臂整机是否整体平移
- 是否到了你想要的作业区域附近

### 步骤 5：再做机械臂轨迹规划

此时再用 MoveIt 或 `moveit_py_demo.py`，去规划机械臂末端动作。

这样做的好处是：

- 先用龙门架做“粗定位”
- 再用机械臂做“精定位”

## 为什么这种方式很有用

因为很多任务里，仅靠机械臂自身 6 轴并不一定能覆盖所有空间。

加入龙门架后，你获得了两层能力：

- 龙门架负责扩大工作空间
- 机械臂负责精细操作

所以龙门架不是机械臂的替代，而是工作空间扩展器。

## 如果你只记住一句话

调整龙门架三轴，本质上是在搬动机械臂的“安装根部”；根部一动，整台机械臂和它的末端工作空间就一起动。
