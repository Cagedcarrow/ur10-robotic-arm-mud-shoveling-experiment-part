# my_robot 龙门架项目总览

## 这篇文档解决什么问题

这篇文档给刚接触 ROS 2、URDF、Gazebo、MoveIt 的读者一个总入口，帮助你先建立全局认识：

- 这次项目最终做成了什么。
- 整个系统由哪些 ROS 2 包组成。
- Gazebo、RViz2、MoveIt、控制器、Xacro 分别负责什么。
- 你应该从哪个启动命令开始用这套系统。

如果你还不熟悉本项目，建议先读这篇，再继续读后面的分篇文档。

## 这次项目最终做成了什么

本次项目把原本单独存在的 `my_robot` 机械臂模型，封装成了一个可复用的 Xacro 宏，然后把它挂接到一个三轴龙门架上，形成了一个“龙门架 + UR10 机械臂 + 铲斗末端”的组合机器人。

组合完成后，又补齐了以下能力：

- Gazebo 中生成整机模型并运行 `ros2_control` 控制器。
- MoveIt 使用新的专用配置包对该组合机器人进行规划。
- RViz2 中可以看到规划场景、机器人状态和交互控制。
- 龙门架三轴能够改变机械臂基座在世界坐标中的位置。
- 机械臂关节轨迹可以通过 MoveIt 下发给 Gazebo 中的控制器执行。

## 你现在能直接怎么启动

先加载 ROS 2 和工作空间环境：

```bash
source /opt/ros/humble/setup.bash
source /root/ur10_ws/install/setup.bash
```

然后启动完整联动系统：

```bash
ros2 launch my_robot sim_planning_gantry.launch.py gui:=true start_rviz:=true
```

这个启动命令会做两件事：

- 启动 Gazebo，生成龙门架与机械臂组合模型，并加载控制器。
- 启动 MoveIt 和 RViz2，让你可以在可视化界面里做规划与观察。

## 项目整体结构

本项目最重要的几个包如下：

- `my_robot`
  - 放机器人本体 Xacro、龙门架 Xacro、Gazebo/控制器配置、启动文件。
- `my_robot_moveit_config`
  - 放这台组合机器人的 MoveIt 配置，包括 SRDF、运动学、控制器映射、RViz 配置和 MoveIt 启动文件。
- `ur10_examples_py`
  - 放 Python 示例脚本，包括通用化后的 `moveit_py_demo.py`、龙门架控制脚本和 RViz 交互控制脚本。
- `docs/my_robot_gantry_build_notes`
  - 放本次项目的说明文档。

## 整体运行链路

可以把系统理解成下面这条链路：

1. Xacro 负责“描述机器人长什么样、关节怎么连接、坐标怎么挂接”。
2. `robot_state_publisher` 负责把机器人模型中的关节关系发布成 TF 坐标树。
3. Gazebo 负责“把这个机器人实体放进物理仿真世界里”。
4. `gazebo_ros2_control` 和 `controller_manager` 负责“把关节控制接口连到 Gazebo 中的仿真机器人”。
5. MoveIt 负责“根据当前机器人状态做运动规划”。
6. RViz2 负责“把 MoveIt 的规划结果、TF、机器人状态和交互控件显示给你”。
7. `moveit_py_demo.py` 或龙门架控制脚本负责“给 MoveIt 或控制器发具体动作命令”。

## 用一句话理解这套系统

这套系统的核心思想是：

同一份 `robot_description` 同时服务于 Gazebo 仿真、TF 发布、MoveIt 规划和 RViz2 显示，这样模型、关节名称、坐标关系和控制接口才能保持一致。

## 这次项目里的关键变化

和之前相比，这次最重要的变化有四类：

1. 机械臂本体从单体描述改成了可复用宏。
2. 在机械臂外面新增了三轴龙门架结构。
3. 新建了专门的 `my_robot_moveit_config` 包来匹配新的机器人结构。
4. 为 Gazebo 增加了更稳定的兼容 visual，避免 Gazebo 中只看到龙门架、看不到 UR10 本体。
