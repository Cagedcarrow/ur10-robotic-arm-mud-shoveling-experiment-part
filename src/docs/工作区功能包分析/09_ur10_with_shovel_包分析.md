# `ur10_with_shovel` 包分析

## 这篇文档解决什么问题

这篇文档解释 `ur10_with_shovel` 为什么还值得保留，以及它和当前 `my_robot` 主线之间是什么关系。

## 一句话说，这个包是干什么的

`ur10_with_shovel` 是“UR10 + 铲斗末端”的专项模型与运行包，重点关注铲斗末端集成、`shovel_tip` 规划和相关运行流程。

## 标签

`历史/专项分支`

## 它在整个工作区中的位置

它不是当前龙门架整机主线，但它很重要，因为它保留了：

- 铲斗末端建模
- 末端 `shovel_tip` 规划约定
- 与铲斗相关的一套 runbook

可以把它理解成：

- 一个历史上非常关键的中间演化版本
- 一个围绕铲斗末端能力的专项分支

## 主要文件和入口说明

### `README.md`

这个包的 README 非常有价值，不只是说明，而是接近运行手册。

它覆盖了：

- 仿真启动
- RViz 规划
- TF 检查
- 控制器检查
- 与实机接口预留相关内容

### `urdf/ur10_shovel.urdf.xacro`

这是本包最核心的模型文件之一。

它定义：

- UR10 + 铲斗
- `shovel_tip`
- 相关末端坐标关系

### `launch/sim_planning_shovel.launch.py`

是本包的仿真一键启动入口。

### `launch/gazebo_shovel.launch.py`

仅 Gazebo 入口。

### `launch/moveit_shovel.launch.py`

仅 MoveIt 规划入口。

### `launch/view_shovel_rviz.launch.py`

纯模型可视化入口。

### `config/ros2_controllers.yaml`

定义本包使用的控制器配置。

## 这个包和其他包的关系

### 和 `ur10_moveit_config` 的关系

它建立在原始 UR10 主线之上。

### 和 `my_robot` 的关系

它不是 `my_robot` 的替代品，但它对理解铲斗末端整合非常有参考价值。

尤其是：

- 铲斗末端 frame
- `shovel_tip`
- 铲斗与机械臂连接关系

这些内容对后来的 `my_robot` 整理是有历史参考价值的。

## 什么时候应该改这个包

如果你现在要做的是：

- 回顾铲斗末端专线
- 对比历史建模方式
- 单独研究 `shovel_tip`

那可以看它。

## 什么时候不应该先改这个包

如果你当前正在维护日常使用的龙门架整机系统，就不要优先改这个包。

当前主线更应该优先看：

- `my_robot`
- `my_robot_moveit_config`

## 新手最容易混淆的点

### 1. 以为它还是当前主线

不是。

它现在更适合被看成专项参考线，而不是默认主入口。

### 2. 以为它已经完全没用了

也不是。

它对“铲斗末端是怎么被接进系统的”这件事仍然很有参考价值。

## 如果你现在只能记住一句话

`ur10_with_shovel` 不是当前整机主线，但它是理解铲斗末端演化历史的重要专项包。
