# UR10 网络通讯包教程（WSL2 Ubuntu 22.04）

本教程面向你的当前环境：**Windows + WSL2 Ubuntu 22.04 + ROS 2 Humble**，目标是把 UR10 实机和电脑连通，并使用 `ur10_real_comm` 完成一次微动闭环验证。

## 1. 教程目标
完成以下闭环：

1. 电脑与 UR10 网络互通；
2. `ur_robot_driver` 能连接机器人并接收回连；
3. MoveIt2 完成 `wrist_3_joint` 微动规划；
4. 人工确认后执行一次微动；
5. `/ur10_real_comm/status` 输出最终 `PASS/FAIL`。

## 2. 先理解网络拓扑（WSL2 关键）
UR 驱动需要机器人回连 PC（`reverse_ip` + 50001~50004 端口）。

在 WSL2 下，推荐使用下面这个稳定方案：

1. UR10 连接到 **Windows 实体网卡**（同网段）；
2. 机器人回连目标设为 **Windows 网卡 IP**（不是 WSL IP）；
3. Windows 使用 `portproxy` 把 50001~50004 转发到 WSL2；
4. 驱动在 WSL2 内运行。

这样即使 WSL2 是 NAT 网络，也能稳定工作。

## 3. 连接前准备

### 3.1 物理连接
1. 用网线连接：UR 控制柜网口 <-> Windows 电脑网口（或同交换机）。
2. 确认网口指示灯正常。

### 3.2 UR10 侧网络参数（示例）
在示教器设置静态 IP（示例）：

1. Robot IP：`192.168.56.101`
2. Subnet mask：`255.255.255.0`
3. Gateway：留空或 `0.0.0.0`

> 你可用别的网段，但下面命令要同步改成同网段。

### 3.3 Windows 网卡静态 IP（示例）
把连接 UR10 的那张 Windows 网卡设为：

1. IPv4：`192.168.56.1`
2. Subnet mask：`255.255.255.0`
3. Gateway：留空

## 4. Windows 端配置（必须）

### 4.1 先测试 Windows 与 UR10 互通
在 Windows PowerShell：

```powershell
ping 192.168.56.101
```

能 ping 通再继续。

### 4.2 获取 WSL2 当前 IP
在 WSL 终端：

```bash
hostname -I | awk '{print $1}'
```

记下结果，例如：`172.30.98.45`。

### 4.3 配置端口转发（管理员 PowerShell）
以下命令将机器人打到 Windows `192.168.56.1` 的回连端口，转发到 WSL：

```powershell
netsh interface portproxy reset

netsh interface portproxy add v4tov4 listenaddress=192.168.56.1 listenport=50001 connectaddress=172.30.98.45 connectport=50001
netsh interface portproxy add v4tov4 listenaddress=192.168.56.1 listenport=50002 connectaddress=172.30.98.45 connectport=50002
netsh interface portproxy add v4tov4 listenaddress=192.168.56.1 listenport=50003 connectaddress=172.30.98.45 connectport=50003
netsh interface portproxy add v4tov4 listenaddress=192.168.56.1 listenport=50004 connectaddress=172.30.98.45 connectport=50004

netsh interface portproxy show all
```

### 4.4 配置 Windows 防火墙放行（管理员 PowerShell）

```powershell
netsh advfirewall firewall add rule name="UR Reverse 50001" dir=in action=allow protocol=TCP localip=192.168.56.1 localport=50001 remoteip=192.168.56.0/24
netsh advfirewall firewall add rule name="UR Reverse 50002" dir=in action=allow protocol=TCP localip=192.168.56.1 localport=50002 remoteip=192.168.56.0/24
netsh advfirewall firewall add rule name="UR Reverse 50003" dir=in action=allow protocol=TCP localip=192.168.56.1 localport=50003 remoteip=192.168.56.0/24
netsh advfirewall firewall add rule name="UR Reverse 50004" dir=in action=allow protocol=TCP localip=192.168.56.1 localport=50004 remoteip=192.168.56.0/24
```

## 5. URCap External Control 配置
在 UR 示教器上：

1. 安装并启用 External Control URCap；
2. 在程序里添加 External Control 节点；
3. Host/IP 填：`192.168.56.1`（Windows 网卡 IP）；
4. Port 填：`50002`（对应 `script_sender_port` 默认值）；
5. 保存并运行该程序节点。

> 如果这里填成 WSL NAT IP，通常会失败。

## 6. WSL2 中运行通讯包验证

### 6.1 编译
```bash
cd /root/ur10_ws
source /opt/ros/humble/setup.bash
colcon build --packages-select ur10_real_comm
source install/setup.bash
```

### 6.2 推荐一键验证（终态判定）
```bash
cd /root/ur10_ws/src/ur10_real_comm/scripts
ROBOT_IP=192.168.56.101 \
REVERSE_IP=192.168.56.1 \
WRIST3_DELTA_DEG=0.5 \
./run_full_verify.sh
```

说明：

1. 脚本会启动完整链路（驱动 + MoveIt + 验证节点）；
2. 监听 `/ur10_real_comm/status`；
3. 首次出现 `PASS/FAIL` 就结束并给出摘要。

### 6.3 人工确认执行（必须）
当状态进入 `WAIT_CONFIRM` 后，在另一个 WSL 终端执行：

```bash
source /opt/ros/humble/setup.bash
source /root/ur10_ws/install/setup.bash
ros2 service call /ur10_real_comm/confirm_execute std_srvs/srv/Trigger "{}"
```

## 7. 成功标准
满足以下即为成功：

1. `/joint_states` 连续更新；
2. 状态流出现 `PLAN -> PLAN_OK -> WAIT_CONFIRM -> EXECUTE -> PASS`；
3. 脚本输出 `[PASS]` 且返回码 `0`。

## 8. 常见故障与排查

### 8.1 机器人连接不上（驱动初始化失败）
检查：

1. Windows 能否 `ping 192.168.56.101`；
2. URCap 程序是否正在运行；
3. `robot_ip` 是否写错。

### 8.2 一直卡在等待或回连失败
重点检查：

1. `reverse_ip` 是否是 Windows 网卡 IP（本教程为 `192.168.56.1`）；
2. `portproxy` 是否存在且指向**当前** WSL IP；
3. Windows 防火墙是否放行 50001~50004。

### 8.3 每次重启 WSL 后又失败
原因：WSL IP 可能变化，端口转发目标失效。

处理：

1. 重新获取 WSL IP；
2. 重新执行 4.3 的 `portproxy` 命令。

### 8.4 规划成功但执行偏差 FAIL
检查：

1. 控制器是否 active：
```bash
ros2 control list_controllers
```
2. 机械臂是否受限位/碰撞环境影响；
3. 日志文件：`/tmp/ur10_real_comm_launch.log`。

## 9. 配置文件对应关系

1. `ur10_real_comm/config/real_robot_network.yaml`
   - 网络参数模板参考（网段、IP、端口）
2. `ur10_real_comm/config/verify.yaml`
   - 微动参数、超时、阈值
3. `ur10_real_comm/launch/real_comm_moveit_verify.launch.py`
   - 单入口总启动
4. `ur10_real_comm/scripts/run_full_verify.sh`
   - 一键执行与终态汇总

## 10. 建议的现场操作顺序（最省时间）
1. 先做 Windows 与 UR10 `ping` 测试；
2. 配好并确认 `portproxy show all`；
3. 启动 URCap External Control 程序；
4. 运行 `./run_full_verify.sh`；
5. 出现 `WAIT_CONFIRM` 后调用确认服务；
6. 记录最终 `PASS/FAIL` 与原因。

## 11. 一体化 WSL Bash 一键脚本（无 PowerShell）

脚本文件：

`docs/ur10_real_comm_实机微动验证总结/run_full_verify_wsl.sh`

这个脚本**完全在 WSL 内执行**，目标是：

1. 使用你已固定的主机网卡 IP（reverse IP）作为机器人回连地址；
2. 做一次基本环境检查；
3. 直接执行 `run_full_verify.sh`，按状态话题终态返回 PASS/FAIL。

使用示例：

```bash
chmod +x /root/ur10_ws/src/docs/ur10_real_comm_实机微动验证总结/run_full_verify_wsl.sh
/root/ur10_ws/src/docs/ur10_real_comm_实机微动验证总结/run_full_verify_wsl.sh   --host-ip 192.168.56.1   --robot-ip 192.168.56.101   --wrist-delta 0.5
```

如果你不想启动 rviz：

```bash
/root/ur10_ws/src/docs/ur10_real_comm_实机微动验证总结/run_full_verify_wsl.sh   --host-ip 192.168.56.1   --robot-ip 192.168.56.101   --no-rviz
```

你可以覆写更多参数：

```bash
/root/ur10_ws/src/docs/ur10_real_comm_实机微动验证总结/run_full_verify_wsl.sh   --host-ip 192.168.56.1   --robot-ip 192.168.56.101   --ws-root /root/ur10_ws   --controller scaled_joint_trajectory_controller   --status-timeout 900   --wrist-delta 0.3   --no-confirm
```

说明：
- `--host-ip` 对应你在 Windows 上的有线网卡 IP（如 `192.168.56.1`）。
- 脚本默认直接读取环境变量 `UR10_WINDOWS_HOST_IP`，也可通过参数覆盖。
- 如果你当前网络已经可直接打通，这个脚本会沿用现状直接跑验证；
  如果你使用 NAT + portproxy 模式，仍需在 Windows 侧确认端口转发已生效。


## 12. 图形化界面（Python, Tkinter）

你提到要“更详细/更好看”的检查体验，我补了一个 GUI：

- 脚本：
  `ur10_real_comm/scripts/ur10_real_verify_gui.py`

### 功能
1. 全部参数可视化填写
   - host-ip（Windows 主机 IP）
   - robot-ip
   - robot-subnet
   - ws-root
   - wrist3_delta_deg
   - controller
   - status-timeout
   - 是否启动 rviz
   - 是否需要 confirm
2. 点击“检查参数与网络”做基础校验
   - IP/Subnet 合法性校验
   - WSL 端口脚本路径与工作区检查
   - 简单 ping 检查（可达性提示）
3. 一键启动验证（内部调用 `run_full_verify_wsl.sh`）
4. 实时日志滚动显示
5. 最终 `[SUMMARY]` 自动提取到状态栏（PASS/FAIL）
6. 提供按钮：
   - 发送确认服务（`ros2 service call ...`）
   - 查询控制器状态（`ros2 control list_controllers`）
7. 支持 Dry Run（仅打印将执行命令，不真的发起验证）
8. 支持参数自动保存到 `~/.ur10_real_verify_gui.json`

### 运行（纯 WSL）

```bash
cd /root/ur10_ws/src/ur10_real_comm
python3 scripts/ur10_real_verify_gui.py
```

### 注意
- GUI 是 WSL 内运行的，不依赖 PowerShell。
- 如果你是 Windows 远程访问该仓库（通过 \wsl$），仍可在 WSL 里直接启动 GUI。
- 验证的核心判定仍由 `run_full_verify.sh` 的状态终态决定，界面只是做可视化封装。
